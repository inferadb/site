We built this market category

InferaDB's team previously built Auth0/Okta's fine-grained authorization solution and its open-source counterpart OpenFGA — now a CNCF incubating project used in production by Grafana Labs, Docker, and Canonical.

Authorization — the system that governs who can access what within software applications — is a hidden crisis in modern enterprise software. Authorization vulnerabilities are the #1 risk in the OWASP API Security Top 10. Ninety-four percent of applications tested exhibit broken access control. Organizations that build authorization in-house typically dedicate six or more engineers to maintaining it, representing roughly $900,000 per year in engineering budget alone.

We experienced this problem firsthand while building Auth0/Okta's authorization products. We identified fundamental architectural limitations in existing approaches — reliance on general-purpose databases, cache-dependent policy engines, and application-level multi-tenancy — and set out to build what we wished we could have built at Auth0: a true authorization database designed from the ground up for this workload.

The Problem

The rise of AI agents is exponentially increasing the number of fine-grained authorization decisions required per request. Traditional authorization systems were built for 1–2 permission checks per request. AI agent workflows require dozens. Meanwhile, three major EU regulations — NIS2, DORA, and the EU AI Act — are converging to mandate fine-grained, auditable access controls. The authorization infrastructure required to meet these demands does not exist in most organizations.

Our Approach

InferaDB is not another policy engine — it's a purpose-built authorization database. Competitors either bolt authorization onto general-purpose databases or operate as policy engines without persistent storage. InferaDB's custom storage engine, cryptographic isolation, and consensus replication represent a fundamentally different architectural approach.

The result is sub-microsecond authorization decisions (2.8µs p99), cryptographic multi-tenant isolation at the storage layer, and tamper-proof audit trails with Merkle proof verification — capabilities that no existing product offers together.

Open Source

InferaDB's core authorization engine is open source, dual-licensed under MIT and Apache 2.0. We believe authorization infrastructure should be transparent, auditable, and community-driven. The managed cloud service adds multi-tenant isolation, cryptographic audit trails, SLAs, and compliance reporting.

Standards

InferaDB implements the OpenID AuthZEN specification — the industry's first standardized authorization API — ensuring vendor-neutral interoperability. Our architecture is inspired by Google Zanzibar, the internal system Google built to handle authorization for YouTube, Drive, and Cloud at trillions of checks per day.
