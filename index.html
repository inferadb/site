Zanzibar-Inspired

Built on Google's proven consistency model with revision tokens for causal ordering, solving the "new enemy problem" that makes eventual consistency unacceptable for authorization.

AuthZEN-Compliant

First-class support for the OpenID Foundation's authorization API specification, enabling vendor-neutral interoperability and zero lock-in.

Open Source

Built for
developers

Authorization should be understandable, testable, and observable. InferaDB provides the tooling to match.

terminal 
$ inferadb policy branch feature/new-rule
Branch created: feature/new-rule

$ inferadb simulate \
    --resource document:1 \
    --subject user:evan
✓ ALLOWED via: owner → edit → view
  Revision: r_2026031400042
  Latency:  2.3ms

$ inferadb merge feature/new-rule
Merged feature/new-rule → main

CLI

Branch, simulate, and merge policies like code. Debug authorization decisions with full trace explanations.

check.rs 
use inferadb::Client;

let client = Client::connect("localhost:9090").await?;

let allowed = client
    .check("document:1", "view", "user:evan")
    .await?;

assert!(allowed.permitted);
