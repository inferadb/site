Traditional authorization systems were designed for a world where each user request triggers one or two permission checks. A user clicks a button, the backend verifies they have access, and the page loads. That model held for twenty years. AI agents have broken it. A single agent action — retrieving context, selecting tools, executing a multi-step workflow — can require dozens of authorization checks in sequence. The agent needs permission to access a knowledge base, permission to read specific documents within it, permission to invoke each tool in its chain, and permission to act on behalf of the user who initiated the request. When each of those checks takes 5 to 50 milliseconds through a traditional authorization engine backed by PostgreSQL, the cumulative latency becomes untenable. Twenty checks at 25 milliseconds each adds half a second of authorization overhead alone, and that is before the agent has done any actual work.

The math gets worse as agent architectures grow more sophisticated. Retrieval-augmented generation pipelines need to authorize access to every document candidate before feeding context to the language model. An agent with access to ten tools might need to verify permissions for each tool invocation, and tools that operate on collections need per-item authorization. At enterprise scale, where a single user might have access to thousands of documents through a complex hierarchy of group memberships, role assignments, and sharing relationships, the authorization graph traversal alone can dominate the latency budget. Systems that rely on caching to mask this latency face an even harder problem: agent workflows are dynamic and contextual, which means cache hit rates drop precisely when authorization load increases.

InferaDB was designed with this workload in mind. At 2.8 microseconds p99 read latency, a sequence of twenty authorization checks completes in under 60 microseconds — roughly a thousand times faster than the same sequence through a cache-miss path on a traditional engine. This is not an incremental improvement that makes agents slightly more responsive. It is a qualitative change that makes per-action, per-document, per-tool authorization feasible in latency-sensitive agent workflows. RAG pipelines can authorize every document candidate at retrieval time rather than relying on post-retrieval filtering, which is both more secure (the unauthorized documents never enter the model's context window) and more efficient (the retrieval layer can prune unauthorized results before ranking).

The security implications extend beyond latency. AI agents operate with delegated authority — they act on behalf of a user, but their behavior is non-deterministic and difficult to predict. Traditional authorization models assume that the caller is a human who understands what they are requesting. Agents do not have that understanding. They follow tool-use patterns that emerge from prompt engineering and model behavior, not from deliberate user intent. This means the authorization layer must enforce tighter boundaries: not just "can this user access this resource" but "can this agent, acting on behalf of this user, invoke this specific tool with these specific parameters at this point in the workflow." Per-action permission scoping at this granularity requires an authorization system that can evaluate complex policies quickly enough to be called on every action without becoming a bottleneck.

Regulation is catching up to this reality. The EU AI Act, with its high-risk provisions taking effect between August 2026 and 2027, mandates human oversight mechanisms, logging requirements, and access controls for AI systems making consequential decisions. High-risk AI systems must demonstrate that humans can intervene, that decisions are auditable, and that access to the system's capabilities is properly controlled. These are not aspirational guidelines — they are legal requirements with significant penalties for non-compliance. An authorization system that can enforce per-action permissions, maintain a cryptographic audit trail of every check, and provide tamper-evident proof of what an agent accessed and why is not a nice-to-have in this regulatory environment. It is a prerequisite for deploying AI agents in any regulated industry.
