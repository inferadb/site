SEC.01 The Problem

Agent authorization is a different workload

A human user makes a few API calls per interaction. An AI agent can make dozens of tool calls, each requiring fine-grained permission checks against the delegating user's access. When your authorization layer adds 5-50ms per check, agent workflows become unusably slow — or dangerously unchecked.

Dozens of checks per request

An agent reasoning over a user's documents might check permissions on 20+ resources in a single turn. Traditional policy engines turn this into 100ms+ of latency.

User-scoped delegation

Agents act on behalf of users. Every action must be constrained to what the delegating user can access — not what the agent's service account allows.

Auditability requirements

When an agent accesses sensitive data, you need to prove why it was allowed. "The AI did it" is not an acceptable answer for compliance.

SEC.03 Per-User RAG Scoping

Retrieval-augmented generation with access control

When your agent retrieves documents for context, it should only see what the requesting user can see. InferaDB's permission-aware search returns only authorized results — no post-filtering, no data leaks through context windows.

SEC.04 Agent Tool Authorization

Fine-grained control over what agents can do

Model which tools each agent can invoke, scoped to the delegating user's permissions. An agent with access to the "send email" tool can only send as the user who authorized it — and only to recipients the user has permission to contact.

Tool-level permissions

Define which tools each agent type can access. Restrict sensitive operations (delete, transfer, admin) to specific agent configurations.

Delegated access scoping

Agents inherit and are constrained by the permissions of the user who invoked them. No privilege escalation through agent indirection.

Real-time revocation

Revoke a user's access and every agent acting on their behalf loses access immediately. Not eventually. Immediately.

SEC.05 Permission Traces

Chain-of-thought permission traces for every agent action

Every authorization decision includes a full resolution trace — which relationships were traversed, which conditions were evaluated, why the decision was allowed or denied. When an agent accesses sensitive data, you can reconstruct exactly why it was permitted.
